## Setup

Install dependencies and run the app:

```bash
npm install
npm run dev
```

Open http://localhost:3000.

## Environment Variables

Add these values to `.env`:

```env
OPENAI_API_KEY=your_openai_api_key_here
OPENAI_MODEL=gpt-4o-mini
OPENAI_IMAGE_MODEL=gpt-image-1
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url_here
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key_here
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key_here
```

`SUPABASE_SERVICE_ROLE_KEY` is used only on the server in the `/api/comics` route.

## Create User Flow

The app now includes a `/create-user` page and a `/api/clients` route.

Submitting the form:

1. Creates a Supabase Auth user.
2. Inserts a matching record into `public.clients`.

The `public.clients.id` column must continue to reference `auth.users(id)` for this flow to work.

## Supabase Table

Create this table in the Supabase SQL editor:

```sql
create table if not exists public.comics (
	id bigint generated by default as identity primary key,
	title text,
	hero_name text,
	hero_type text,
	reader_age integer,
	genre text,
	visual_style text,
	picture_name text,
	generated_concept text not null,
	cover_image_data_url text,
	created_at timestamptz not null default now()
);
```

## Current Supabase Flow

The comic generator now:

1. Calls the OpenAI text route.
2. Calls the OpenAI image route.
3. Sends the full generated result to `/api/comics`.
4. Inserts the record into the Supabase `comics` table.

If the comic generates successfully but the database insert fails, the UI shows a separate save error.
